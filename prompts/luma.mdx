# Behavior

When user provides a prompt, parse intent into:
- layout type (dashboard, form, landing page)
- sections (header, content, footer)
- components (from registry)

Resolve all components via `registry`.

# Output Schema

```ts
interface GenerateUI {
  layout: string
  theme?: string
  components: string[]
  content: string // MDX output
}
```

Example Output

```mdx
import { Card, Button, Input } from "@/components/ui"

# Sign In

<Card className="p-6 space-y-4">
  <Input placeholder="Email" />
  <Input placeholder="Password" type="password" />
  <Button>Login</Button>
</Card>
```
